Event Channel in Juwai.com 

Overview

Juwai runs regular events for Chinese buyers to learn about the property market. However, the participation rate was not ideal. So our marketing team suggested to develop an individual Channel in Juwai.com to promote the events as well as boosting the participation rate.

Role: 90% UX Research，100% UI Design

Scope

We outlined the current problems that our marketing team was encountering to define the scope of the platforms which are Event Channel, the new channel on Juwai.com and its CMS for publish and manage event promotional webpages. We listed out the scope on a table as below, we always refer to this table during the design process, so we didn't get lost.

In summary, Event Channel is a new channel on Juwai.com, will show past and future events and direct users to Event Details pages(landing page) respectively. Users could learn about the event details and apply for it. The CMS is for our marketing team to edit and publish Event Details page, it will be added to our administrative panel. 

Study the existing event promotion

After defined the scope, I did a research on existing event posters and landing pages to find out what information should be presented for promoting an event. Basically, there are 2 main event types: live broadcast, which do not need user to register but show a QR code for them to join the event; and offline event e.g. conference, seminar, exhibition, which prefer user to register by filling a form. Also, all event details page will present different content based on the event topic.

Besides, I realised the current event details landing pages were actually a page embedded a poster and a digital form. However, more than 70% of user are using mobile to access the page, according to our user data. They might not be able to view the text on the poster. Some posters were too long with large file size, that led to long loading time. So our event details landing page should be optimised to both desktop view and mobile view. 

Wireframe

After that I started to draft out wireframe of Event Channel. 

Event Channel Homepage
At first, the Marketing team suggested to show all events in a timeline view. I designed a wireframe and show it to different department, and quickly got feedback. As a result, people thought the timeline was not very intuitive and wasted a lot of space. In the end, we decided to use the grid layout.

Event Details Page

The responsive landing page was divided into 9 sections to introduce an event:

1. Banner
2. Name/Venue/Date
3. Opening Intro
4. Detailed Intro
5. Photo Album
6. Speakers/Guests
7. Schedule
8. Organisers
9. Press Media

These sections were designed as separated modules in CMS with predefined visual design and able to be show or hide on the page. 


Responsive

The channel is responsive so it adapts to all size of device. 

CMS

A event management tab is added to our administration panel where marketing team manage the event details landing pages

Event Details Landing Page List

The team could create a new event page and manage the existing pages in the list. Each event page has the colour-coded event status  i.e. open for registration, end of event, and unpublished.

Event Landing Page Edit Mode

An event landing page could be created without coding.

Landing Page Modules in CMS

Marketing team can control the appearance of each module(show or hide), edit text and upload images to the predefined input fields in CMS.

All edits will be saved automatically as draft, click the top right “publish” corner to publish the page which will be then generated and showed up on Event Channel.
